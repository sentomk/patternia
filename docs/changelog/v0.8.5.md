# Patternia v0.8.5 Release Note

**Release Date:** March 10, 2026  
**Version:** 0.8.5

---

## Overview

Patternia v0.8.5 is an API modernization release that introduces variable templates
for type patterns and new structural pattern capabilities. All additions are
backward-compatible; existing code continues to compile without changes.

---

## Highlights

### Type Pattern Variable Templates

Type patterns now use modern C++ variable templates instead of the `type::` namespace:

```cpp
// Before
match(v)
  .when(type::is<int>() >> [] { /* ... */ })
  .when(type::as<std::string>() >> [](const std::string &s) { /* ... */ })
  .otherwise([] {});

// After
match(v)
  .when(is<int>() >> [] { /* ... */ })
  .when(as<std::string>() >> [](const std::string &s) { /* ... */ })
  .otherwise([] {});
```

The new variable templates are:
- `is<T>()` - type-only match (replaces `type::is<T>()`)
- `as<T>()` - explicit binding shorthand (replaces `type::as<T>()`)
- `alt<I>()` - index-based variant dispatch (replaces `type::alt<I>()`)

### Destructure-and-Bind Pattern: `ds<>`

A new `ds<&T::m...>()` pattern combines structural validation with automatic binding:

```cpp
struct Point { int x, y; };

match(p)
  .when(ds<&Point::x, &Point::y>() >> [](int x, int y) {
    return x + y;
  })
  .otherwise([] {});
```

`ds<>` is equivalent to `bind(has<>())` but more concise.

### Guarded Structural Validation: `has<>()[guard]`

Structural patterns can now be guarded without binding:

```cpp
match(p)
  .when(has<&Point::x, &Point::y>()[arg<0> + arg<1> == 0] >> [] {
    // Validates structure and guard, but does not bind
  })
  .otherwise([] {});
```

This enables validation-only patterns where binding is not needed.

## Deprecations

- `type::is<T>()`, `type::as<T>()`, and `type::alt<I>()` are deprecated.
  Use the new variable templates `is<T>()`, `as<T>()`, and `alt<I>()` instead.
- The `type::` namespace will be removed in v0.9.0.

## Internal

- Added `is_factory`, `as_factory`, and `alt_factory` wrapper types with
  `binding_args` trait specializations for eval/optimize engine integration.
- Implemented `invoke_guard` helper for mixed tuple/callable guard evaluation.
- Added `has_guarded_pattern` support for structural validation with guards.
- Implicit `lit()` wrapping via `operator>>` SFINAE overload for convenience.
- Migrated all tests, samples, and benchmarks to use new variable templates.
- Updated API documentation to reflect new syntax.

## Migration Guide

### Type Patterns

Replace `type::` namespace calls with variable templates:

```cpp
// Old
type::is<T>()
type::as<T>()
type::alt<I>()

// New
is<T>()
as<T>()
alt<I>()
```

### Structural Binding

Use `ds<>` for concise destructure-and-bind:

```cpp
// Old
bind(has<&T::x, &T::y>())

// New
ds<&T::x, &T::y>()
```

### Structural Validation

Use `has<>()[guard]` for validation without binding:

```cpp
// Old (not possible)

// New
has<&T::x, &T::y>()[arg<0> + arg<1> == 0]
```
