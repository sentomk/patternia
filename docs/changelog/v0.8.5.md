# Patternia v0.8.5 Release Note

**Release Date:** March 10, 2026  
**Version:** 0.8.5

---

## Overview

Patternia v0.8.5 is an API modernization release that introduces variable templates
for type patterns, callable `$(pattern)` syntax for explicit binding, and enhanced
structural pattern capabilities. This release improves API consistency by making
binding intent more explicit and uniform across all pattern types.

---

## Highlights

### Type Pattern Variable Templates

Type patterns now use modern C++ variable templates instead of the `type::` namespace:

```cpp
// Before
match(v)
  .when(type::is<int>() >> [] { /* ... */ })
  .otherwise([] {});

// After
match(v)
  .when(is<int>() >> [] { /* ... */ })
  .otherwise([] {});
```

The new variable templates are:
- `is<T>()` - type-only match (replaces `type::is<T>()`)
- `alt<I>()` - index-based variant dispatch (replaces `type::alt<I>()`)

### Callable `$(pattern)` Syntax for Explicit Binding

The `$` binding pattern is now callable, enabling a more uniform syntax for explicit binding:

```cpp
struct Point { int x, y; };

// Bind variant alternative
match(v)
  .when($(is<int>()) >> [](int i) { return i * 2; })
  .otherwise([] {});

// Bind structural members
match(p)
  .when($(has<&Point::x, &Point::y>()) >> [](int x, int y) {
    return x + y;
  })
  .otherwise([] {});
```

**Design rationale:**
- `$(pattern)` makes binding intent explicit and uniform across all pattern types
- Equivalent to `bind(pattern)` but more concise
- `$` itself remains a binding pattern (equivalent to `bind()`)
- `$(pattern)` can be guarded: `$(is<int>())[_0 > 0]`

### Guarded Structural Validation: `has<>()[guard]`

Structural patterns can now be guarded without binding:

```cpp
match(p)
  .when(has<&Point::x, &Point::y>()[arg<0> + arg<1> == 0] >> [] {
    // Validates structure and guard, but does not bind
  })
  .otherwise([] {});
```

This enables validation-only patterns where binding is not needed.

## Deprecations

- `type::is<T>()` and `type::alt<I>()` are deprecated.
  Use the new variable templates `is<T>()` and `alt<I>()` instead.
- The `type::` namespace will be removed in v0.9.0.

## Internal

- Implemented `bind_factory` type that makes `$` callable with `operator()()`
- Added `has_alt_index` trait to detect type patterns for special binding behavior
- Enhanced `binding_as_pattern` to extract variant alternatives when binding type patterns
- Updated `binding_args` trait to correctly handle `$(is<T>())` binding semantics
- Implemented `invoke_guard` helper for mixed tuple/callable guard evaluation
- Added `has_guarded_pattern` support for structural validation with guards
- Implicit `lit()` wrapping via `operator>>` SFINAE overload for convenience
- Migrated all tests, samples, and benchmarks to use new syntax
- Updated API documentation to reflect new patterns

## Migration Guide

### Type Patterns

Replace `type::` namespace calls with variable templates:

```cpp
// Old
type::is<T>()
type::alt<I>()

// New
is<T>()
alt<I>()
```

### Binding Variant Alternatives

Use `$(is<T>())` for explicit binding:

```cpp
// Old (if you used type::as<T>())
type::as<T>()

// New
$(is<T>())
```

Use `$(has<>())` for explicit binding:

### Structural Binding

Use `$(has<>())` for explicit binding:

```cpp
// Old
bind(has<&T::x, &T::y>())

// New (more concise)
$(has<&T::x, &T::y>())
```

### Structural Validation with Guards

Use `has<>()[guard]` for validation without binding:

```cpp
// New capability in v0.8.5
has<&T::x, &T::y>()[arg<0> + arg<1> == 0]
```
